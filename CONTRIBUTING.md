# Contributing to POS Ledger NG

First of all, thank you for your interest in contributing to **POS Ledger NG**.

We welcome contributions from developers, designers, testers, technical writers, students, fintech professionals, and anyone interested in improving financial technology for small businesses and POS agents.

Every contribution helps make the project better.

---

# Our Mission

POS Ledger NG aims to provide simple, reliable, and accessible financial management tools for POS agents, mobile money operators, and small businesses across Nigeria and Africa.

We believe open collaboration leads to stronger, more useful software.

# Getting Started

1. Fork this repository.
2. Clone your fork to your local machine.

```bash
git clone https://github.com/YOUR_USERNAME/POS-Ledger-NG.git
```

3. Navigate into the project.

```bash
cd POS-Ledger-NG
```

4. Install project dependencies.

```bash
pip install -r requirements.txt
```

5. Create a new branch for your changes.

```bash
git checkout -b feature/your-feature-name
```

6. Make your changes.

7. Test your changes before submitting.

8. Commit your work.

```bash
git commit -m "Add: Short description of changes"
```

9. Push your branch.

```bash
git push origin feature/your-feature-name
```

10. Open a Pull Request on GitHub.

# Development Guidelines

Please follow these principles:

- Keep code clean and readable.
- Write descriptive variable names.
- Add comments where necessary.
- Follow Python best practices.
- Keep functions focused on a single responsibility.
- Avoid unnecessary dependencies.
- Preserve backward compatibility whenever possible.

---

# Coding Standards

Python code should:

- Follow PEP 8 style guidelines.
- Use meaningful function names.
- Use descriptive variable names.
- Include docstrings for public functions where appropriate.
- Handle errors gracefully.
- Validate user input.

---

# Reporting Bugs

If you discover a bug, please include:

- Operating System
- Python version
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if applicable)
- Error messages

# Pull Request Guidelines

Before submitting a Pull Request, please ensure:

- Your code builds successfully.
- Existing functionality is not broken.
- New features are tested.
- Documentation is updated if necessary.
- Commit messages are clear and descriptive.

Example:

```
Add transaction search feature

Fix CSV export formatting

Improve dashboard calculations
```
