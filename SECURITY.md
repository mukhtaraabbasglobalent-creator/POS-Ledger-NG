# Security Policy

Thank you for helping keep **POS Ledger NG** secure.

Security is an important part of our mission to provide reliable financial management tools for POS agents, mobile money operators, and small businesses. We appreciate responsible security research and encourage the responsible disclosure of potential vulnerabilities.

---

# Supported Versions

The project is currently in the **Minimum Viable Product (MVP)** stage.

Security updates will be applied to the latest version of the project as development continues.

| Version | Supported |
|---------|-----------|
| Latest (Main Branch) | ✅ Yes |
| Older Versions | ❌ No |

---

# Reporting a Security Vulnerability

If you discover a security vulnerability, please report it responsibly.

Please include:

- A description of the vulnerability
- Steps to reproduce the issue
- The potential impact
- Screenshots or proof of concept (if appropriate)
- Suggested mitigation (optional)

Please avoid publicly disclosing vulnerabilities before they have been reviewed.

---

# Current Security Measures

The current MVP includes:

- Local SQLite database storage
- Structured transaction recording
- Input validation for financial data
- Modular application architecture
- Local report generation
- Reduced manual calculation errors

---

# Planned Security Enhancements

Future versions of POS Ledger NG aim to include:

- User authentication
- PIN-based login
- Role-based access control
- Database encryption
- Secure password hashing
- Automatic backups
- Cloud synchronization with encryption
- Audit logging
- Multi-user permissions
- Secure API authentication
- Session management
- Data integrity verification

These features are part of the product roadmap and are not yet implemented in the current MVP.

---

# Security Best Practices for Users

Users are encouraged to:

- Keep regular backups of business records.
- Protect devices with a strong password or PIN.
- Restrict access to authorized users.
- Install software updates when available.
- Store exported reports securely.

---

# Responsible Disclosure

We support responsible disclosure and ask security researchers to:

- Act in good faith.
- Avoid disrupting services or deleting data.
- Give us reasonable time to investigate reported issues before public disclosure.

---

# Scope

This security policy applies to:

- Application source code
- Documentation
- Database structure
- Financial calculations
- Report generation
- Future API integrations

---

# Security Roadmap

Our long-term security goals include:

- Secure authentication
- End-to-end encrypted cloud backups
- Compliance with financial security best practices
- Improved audit logging
- Secure software development lifecycle
- Regular security reviews and testing
