# Code of Conduct

## Our Commitment

At **POS Ledger NG**, we are committed to fostering an open, welcoming, inclusive, and respectful community for everyone.

We believe that collaboration, professionalism, and mutual respect are essential to building high-quality financial technology that benefits businesses across Nigeria and Africa.

All contributors, maintainers, users, and community members are expected to follow this Code of Conduct.

---

# Our Values

We value:

- Respect
- Integrity
- Collaboration
- Professionalism
- Transparency
- Inclusiveness
- Continuous Learning
- Constructive Feedback

Everyone should feel welcome regardless of:

- Experience level
- Technical background
- Education
- Nationality
- Gender
- Age
- Language
- Disability
- Religion
- Personal identity

---

# Expected Behavior

Examples of positive behavior include:

- Being respectful and courteous.
- Welcoming new contributors.
- Providing constructive feedback.
- Listening to different perspectives.
- Sharing knowledge openly.
- Helping others learn.
- Reporting issues responsibly.
- Respecting project decisions.
- Writing clear and professional communication.
- Supporting a collaborative environment.

---

# Unacceptable Behavior

The following behaviors are unacceptable:

- Harassment or discrimination.
- Personal attacks.
- Hate speech.
- Offensive language.
- Bullying or intimidation.
- Trolling or deliberate disruption.
- Sharing private information without permission.
- Fraudulent or deceptive activities.
- Plagiarism or copyright infringement.
- Submitting malicious code.
- Any illegal activity.

---

# Contribution Standards

All contributions should:

- Improve the project.
- Follow project documentation.
- Be respectful of existing work.
- Include clear descriptions.
- Follow coding standards.
- Avoid unnecessary complexity.
- Be tested where possible.

---

# Communication Guidelines

When participating in discussions:

- Be respectful.
- Stay professional.
- Focus on solving problems.
- Assume good intentions.
- Welcome different viewpoints.
- Keep discussions constructive.

Healthy disagreement is acceptable, but personal attacks are not.

---

# Reporting Issues

If you experience or witness unacceptable behavior:

- Report it to the project maintainer.
- Include relevant details.
- Provide screenshots or evidence if available.
- Allow maintainers time to investigate.

All reports will be reviewed fairly and confidentially where possible.

---

# Enforcement

Project maintainers are responsible for enforcing this Code of Conduct.

Actions may include:

- Friendly reminder
- Request to modify behavior
- Removal of inappropriate content
- Temporary restriction from contributing
- Permanent removal from the project community

Enforcement decisions will be made with the goal of maintaining a respectful and productive environment.

---

# Scope

This Code of Conduct applies to all project spaces, including:

- GitHub repository
- Issues
- Pull Requests
- Discussions
- Documentation
- Community communications
- Future collaboration platforms
